胡言乱语

每次用git都要现查各个指令，用的晕晕乎乎的，到现在连git是什么也不知道，索性就自己写一个Blog，记录一下git的学习历程吧，这样就可以查自己的blog了(。

什么是git？

Git 是一个开源的 分布式 版本控制 系统，用于敏捷高效地处理任何或小或大的项目。

版本控制

项目开发过程中，增增减减会产生多个不同的版本，项目较小且版本较少时我们可以简单地通过备份多个版本的方法完成版本控制，但当项目规模增大，版本数量开始膨胀时，采用原始地备份方法显然已经难以满足需求，此时就轮到git出场了，利用git可以轻松地回溯版本，完成不同版本的合并。

分布式

  • 中心式：所有项目代码存在一个服务器上，所有人都在这个服务器上开发。
  • 分布式：每个人都可以在本地获得一个项目及其历史的镜像，在本地完成开发，然后在服务器汇总。
  • 中心式一旦服务其宕机就无法进行开发，分布式则规避了这一情况。
  • 由于可以有多个远程库，分布式开发协作起来更加方便，各个团队可在不同分支开发，最后汇总。

git的结构

三种状态

  • Modified: 文件做了修改，但尚未提交至本地的数据库
  • Staged: 修改后的文件已被标记，将在下一次commit时提交至本地数据库
  • Committed: 文件已被安全提交至本地数据库

    三个概念

  • Working Directory/Tree: 简单地说就是你用IDE看到的这个版本，即目前正在开发的版本
  • Staging Area (Index): 是一个文件，记录staged的信息
  • .git Directory (Repository): .git文件夹，存储项目信息。

git的工作流程

  • 在 Working Tree 上修改文件
  • 选择想要commit的标记为staged，记录在Index中
  • 提交Staging Area中的文件，该分支的最新快照将记录在本地Repository中
